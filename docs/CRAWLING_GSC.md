# Google Search Console – Crawling

## How crawling is set up

- **robots.txt** is generated by `app/robots.ts` (no static file in `public/`). It:
  - Allows all user agents on `/`
  - Disallows `/api/` and `/admin/`
  - Declares the sitemap: `https://lemah.store/sitemap.xml`

- **Sitemap index** at `https://lemah.store/sitemap.xml` lists:
  - `sitemap-pages.xml` – /, /shop, /about, /contact, /blog, /search, /products (noindex routes like /11kit excluded)
  - `sitemap-collections.xml` – collection search URLs
  - `sitemap-products.xml` – product pages from Shopify
  - `sitemap-blogs.xml` – blog posts from `content/blog`

- **Canonical domain** for sitemaps and metadata is `https://lemah.store` (from `content/seo/seo-map.json`).

## Verifying in Google Search Console

1. Add the property for `https://lemah.store` (or your live domain).
2. **Sitemaps:** Submit `https://lemah.store/sitemap.xml`. GSC will discover the child sitemaps.
3. **URL Inspection:** Test a few URLs (e.g. `/`, `/shop`, `/about`) to confirm “URL is on Google” or “URL is not indexed” and that there are no blocking issues.
4. **Pages:** Use “Pages” to see indexed vs not indexed; fix any “Crawled – currently not indexed” if you want those pages indexed.
5. **Robots.txt:** In GSC, open “robots.txt” under Settings to confirm it shows the generated file and the `Sitemap:` line.

## Removed for performance / cleanliness

- Static `public/robots.txt` was removed so the dynamic `app/robots.ts` (with sitemap) is always used.
- `/api/test-shopify` route was removed; `/api/` is already disallowed in robots.
